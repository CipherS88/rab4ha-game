## 2. حالات التعادل المعقدة (Ties & Jabir)

### 2.1 تعادل الأبناط تماماً (مثلاً 106 مقابل 106)

- الكود يقارن نقاط اللوحة بعد التحويل: `bScore` (المشتري) vs `dScore` (المدافع).
- شرط الفوز: **`bScore > dScore` فقط** (أكبر strictly — التعادل لا يكفي).
- **عند التعادل (`bScore === dScore`):**
  - يُعتبر **سقوطاً للمشتري** (`is_fall = true`).
  - **المشتري:** 0.
  - **المدافع:** قاعدة السقوط الثابتة + مشاريع المدافع فقط (**30** صن / **16** حكم).
  - **لا يحصل المدافع على أبناط أكلاته** — فقط الثابت + مشاريعه.

### 2.2 هل يوجد "جبر الستة"؟

- **لا.** لا يوجد في الكود أي شرط يخصم 5 أبناط من المشتري ويضيفها للمدافع عند التعادل.
- الموجود فقط: **التجبير** `(abnat + 5) / 10` عند التحويل للوحة — وهذا **ليس** "جبر الستة" القانوني.

### 2.3 التعادل مع التدبيل (دبل / ثري / فور)

- الفوز للمشتري: `bScore > dScore` (strict).
- **عند التعادل:** المشتري **يخسر** (سقوط).
  - **المشتري:** 0.
  - **المدافع:** `16 × مستوى التدبيل + مشاريع المدافع` (بدون أبناط أكلاته).
- **المدبل يُخسر تلقائياً** عند التعادل أو أي حالة لا يكون فيها `bScore > dScore`.

| مستوى التدبيل | نقاط المدافع عند سقوط المشتري (بدون مشاريع) |
|---------------|---------------------------------------------|
| دبل (2) | 32 |
| ثري (3) | 48 |
| فور (4) | 64 |

### 2.4 التعادل مع القهوة (`double_level === 5`)

- عند التعادل في الأبناط: **`winner = buyer`** (المشتري يُفترض فائزاً).
- الفائز يأخذ **150**، الخاسر **0**.
- **لا يُعتبر سقوطاً** للمشتري عند التعادل في القهوة — بل يفوز المشتري.

### 2.5 حالة خاصة: صن + المشتري فوق 100 (`sun_over100_special`)

- إذا المشتري ≥ 100 والخصم < 100 عند شراء صن → تُفتح مرحلة تدبيل خاصة.
- المدافع يستطيع **دبل فقط** أو **بس**.
- **ليست** قاعدة جبر الستة — مرتبطة بموقف المباراة لا بتعادل الأبناط.

## 3. الصن والحكم العادي

### 3.1 فوز المشتري (`bScore > dScore`)

- **الفريقان** يأخذان نقاطهما الطبيعية بعد التحويل.
- كل فريق = تجبير (أكلاته + أرضه + مشاريعه).
- `is_fall = false`.

### 3.2 خسارة المشتري / السقوط (`bScore <= dScore`، بما فيها التعادل)

- **المشتري:** 0 (تطير أكلاته ومشاريعه).
- **المدافع:** **30** (صن) أو **16** (حكم) + مشاريع المدافع فقط.
- **لا يحصل المدافع على أبناط أكلاته** في حالة السقوط.
- `is_fall = true`.

### 3.3 الأرض (10 أبناط)

- تُمنح لفريق **آكل آخر أكلة** (`last_trick_winner_team`).
- **لا** لمن أكَل أكثر أكلات.

### 3.4 المشاريع في الصن

- تُحسب **لفريق الفائز بالمشاريع فقط** (`winning_project_team`).
- فريق الخصم: مشاريعه = **0** في الحساب (حتى لو أعلنها).

### 3.5 المشاريع في الحكم

- إذا وُجد **بلوت** → **كل** المشاريع المعلنة لجميع اللاعبين تُحسب.
- بدون بلوت → فقط مشاريع **الفريق الفائز بالمشاريع**.
- نقاط البلوت تُضاف منفصلة للفريق الذي أعلنه.

## 4. حالات السوا (Sawa)

### 4.1 السوا الصحيح (`end_reason === 'sawa'`)

1. يُعطى **10 أبناط أرض** لفريق المُعلِن (`forced_win_team`) — **وليس** لآكل آخر أكلة.
2. يُحسب باقي النقاط **بشكل طبيعي** عبر `settlePlayingRound` → `settleNormalRound`.
3. **لا نقاط ثابتة** للمُعلِن — يُحسب مثل جولة عادية + الأرض.

### 4.2 توزيع النقاط في السوا

| الحالة | النتيجة |
|--------|---------|
| المشتري فاز (`bScore > dScore`) | الفريقان يأخذان `base_final` كاملاً (أكلات + أرض + مشاريع) |
| المشتري خسر/تعادل | سقوط — المشتري **0**، المدافع **30/16 + مشاريعه** |
| مشاريع الخصم | تُحسب ضمن `base_final` قبل قرار السقوط؛ عند السقوط **لا تُمنح** للمدافع إلا مشاريعه |

### 4.3 السوا الخاطئ

- يُعامل كـ **قيد** (`end_reason === 'qaid'`).
- الفائز = فريق المعترض.
- **لا أرض** في القيد.

## 5. التدبيل (Doubling)

### 5.1 مستويات التدبيل في الكود

| المستوى | المعنى |
|---------|--------|
| 1 | عادي (بدون تدبيل) |
| 2 | دبل |
| 3 | ثري |
| 4 | فور |
| 5 | قهوة |

### 5.2 المشتري **فاز** في التحدي

- **الفريقان** يأخذان: `تجبير((أكلات + أرض + مشاريع) × المضاعف)`.
- يُضاعف **المجموع الكامل** (أكلات + أرض + مشاريع) ثم يُحوَّل للوحة.
- **ليس** الأكلات فقط.

### 5.3 المشتري **خسر** في التحدي

- **المشتري:** 0.
- **المدافع:** `16 × المضاعف + مشاريع المدافع` (بدون أبناط أكلاته).

### 5.4 القهوة

- الفائز بالأبناط: **150** للفائز، **0** للخاسر.
- **عند التعادل:** المشتري يفوز بـ 150.
- بعد `finalize_round`: تُستبدل النتيجة بـ 150/0 وتُنهى المباراة (`match_winner`).

## 6. الكبوت والقيد

### 6.1 كبوت (8 أكلات) — `tricks_won === 8`

- **يُفحص قبل** أي حساب عادي.
- **الفائز:** 44 (صن) أو 25 (حكم) + مشاريع الفائز.
- **الخاسر:** 0.
- `is_kaput = true`.

### 6.2 قيد صحيح (`end_reason === 'qaid'`)

#### الصن

| الفريق | النقاط |
|--------|--------|
| الفائز (المُقيِّد) | **30 + مشاريع الفائز** |
| الخاسر | **0** |

- إذا الخاسر **0 أكلات** → `is_kaput = true` (علم فقط، النقاط ثابتة 30).

#### الحكم العادي (`double_level = 1`)

| الفريق | النقاط |
|--------|--------|
| الفائز | **16 + مشاريع الفائز** |
| الخاسر | **0** |

#### الحكم + تدبيل

| المستوى | نقاط الفائز (بدون مشاريع) |
|---------|---------------------------|
| دبل | 32 |
| ثري | 48 |
| فور | 64 |

#### الحكم + قهوة + قيد

| الفريق | النقاط |
|--------|--------|
| الفائز | **150** |
| الخاسر | **0** |

### 6.3 قواعد المشاريع في القيد

- **يُصفَّر** كل مشاريع الخاسر.
- **يُحسب** فقط مشاريع الفائز عبر `_computeQaidProjects`.
- **لا أرض** في القيد (`ground = 0`).

### 6.4 سوا غلط → قيد

- نفس قواعد القيد أعلاه.
- `qaid_loser_tricks` = أكلات فريق المُعلِن.

## 8. تدفق `finalize_round` (مختصر)

```
1. جمع raw_tricks + ground + raw_proj → abnat
2. تحويل abnat → base_final (تجبير)
3. إذا forced_win_team:
   ├─ سوا        → settlePlayingRound (حساب عادي + أرض المُعلِن)
   ├─ قيد        → نقاط قسرية + مشاريع الفائز فقط
   └─ آخر        → كبوت قسري
4. وإلا → settlePlayingRound:
   ├─ 8 أكلات؟   → كبوت
   └─ وإلا       → settleNormalRound (فوز / سقوط / تدبيل)
5. إذا قهوة → استبدال النتيجة بـ 150/0
6. فحص match_winner عند ≥ 152
```

## 10. نقاط التركيز لبرومبت التعديل المستقبلي

1. **جبر الستة** — غير موجود حالياً؛ التعادل = سقوط المشتري مباشرة.
2. **سقوط المدافع** — المدافع لا يأخذ أبناط أكلاته، فقط الثابت + مشاريعه.
3. **تعادل القهوة** — المشتري يفوز 150 تلقائياً.
4. **السوا** — ليس نقاطاً ثابتة؛ حساب طبيعي + 10 أرض للمُعلِن.
5. **التدبيل** — يضاعف المجموع الكامل (أكلات + أرض + مشاريع) عند الفوز.
